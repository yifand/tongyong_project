# PDI智能监测平台 - 团队组建与开发计划

## 一、项目概述

**项目名称**: PDI智能监测平台 - 金桥基地VDC智能算法项目业务平台
**技术栈**: Spring Boot 3.2.x + JDK 17 + PostgreSQL 16 + Redis + Maven
**项目结构**: 多模块Maven项目

### 核心功能模块
1. 预警中心模块（实时预警、历史预警）
2. 行为档案模块（PDI作业档案、时间线）
3. 设备管理模块（边缘盒子、通道管理）
4. 系统管理模块（用户、角色、权限、日志）
5. 数据接收服务（状态流、报警事件、心跳）

## 五、开发规范

### 5.1 代码规范

#### 命名规范
- 包名: 全小写，如 `com.pdi.service.alarm`
- 类名: 大驼峰，如 `AlarmService`
- 方法名: 小驼峰，如 `getAlarmById`
- 常量: 全大写下划线，如 `MAX_RETRY_COUNT`

#### 接口规范
- URL: `/api/v1/{module}/{resource}`
- 响应格式: `{code, message, data, timestamp}`
- HTTP状态码: 200成功，400参数错误，401未认证，403无权限，500服务器错误

### 5.2 Git工作流

```
main（主分支，生产环境）
  ↑
develop（开发分支，日常开发）
  ↑
feature/xxx（功能分支）
```

- 功能开发从 develop 切出 feature 分支
- 功能完成后合并回 develop
- 版本发布时 develop 合并到 main

### 5.3 数据库规范

- 表名: 小写下划线，如 `sys_user`
- 字段名: 小写下划线，如 `created_at`
- 主键: `id` BIGINT 自增
- 时间戳: `created_at`, `updated_at`
- 删除标记: `deleted` SMALLINT DEFAULT 0
