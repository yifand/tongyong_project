# PDI智能监测平台 - API接口规范文档

## 1. API设计规范

### 1.1 URL设计规范

#### 基本格式
```
/api/v{版本号}/{模块}/{资源}/{动作}
```

#### 设计原则
| 原则 | 说明 | 示例 |
|------|------|------|
| 小写字母 | URL全部使用小写 | `/api/v1/alarms` |
| 复数名词 | 资源使用复数形式 | `/api/v1/users` |
| 连字符分隔 | 多单词使用连字符 | `/api/v1/realtime-alarms` |
| 无动词 | URL中不包含动词 | 使用HTTP方法表达动作 |
| 层级关系 | 使用路径表达层级 | `/api/v1/boxes/{id}/channels` |

#### HTTP方法使用
| 方法 | 用途 | 示例 |
|------|------|------|
| GET | 获取资源 | `GET /api/v1/alarms` |
| POST | 创建资源 | `POST /api/v1/boxes` |
| PUT | 全量更新 | `PUT /api/v1/users/{id}` |
| PATCH | 部分更新 | `PATCH /api/v1/alarms/{id}/status` |
| DELETE | 删除资源 | `DELETE /api/v1/channels/{id}` |

#### 版本控制策略
- **URL路径版本**: `/api/v1/`, `/api/v2/`
- **版本升级原则**:
  - 破坏性变更必须升级版本号
  - 新增接口保持当前版本
  - 废弃接口保留至少2个版本周期

### 1.2 请求规范

#### 请求头规范
| 请求头 | 必填 | 说明 |
|--------|------|------|
| Authorization | 是 | Bearer Token认证 `Bearer {accessToken}` |
| Content-Type | 是 | `application/json` |
| X-Site-Id | 条件 | 数据权限站点ID(非超管必填) |
| X-Request-Id | 否 | 请求追踪ID |
| Accept-Language | 否 | 语言偏好 `zh-CN`, `en-US` |

#### 请求示例
```http
GET /api/v1/alarms?page=1&size=20&status=0 HTTP/1.1
Host: api.pdi.example.com
Authorization: Bearer eyJhbGciOiJIUzI1NiIs...
Content-Type: application/json
X-Site-Id: 1
```

### 1.3 响应规范

#### 统一响应格式
```json
{
  "code": 200,
  "message": "success",
  "data": {},
  "timestamp": 1711459200000
}
```

#### 响应字段说明
| 字段 | 类型 | 说明 |
|------|------|------|
| code | Integer | 状态码(200成功, 其他见错误码定义) |
| message | String | 响应消息 |
| data | Object | 响应数据 |
| timestamp | Long | 服务器时间戳(毫秒) |

#### 成功响应示例
```json
{
  "code": 200,
  "message": "操作成功",
  "data": {
    "id": 1,
    "alarmType": 1,
    "alarmTitle": "PDI作业时长不足"
  },
  "timestamp": 1711459200000
}
```

#### 分页响应格式
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "list": [...],
    "total": 100,
    "pages": 5,
    "pageNum": 1,
    "pageSize": 20
  },
  "timestamp": 1711459200000
}
```

#### 分页参数规范
| 参数 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| page | Integer | 否 | 1 | 当前页码(从1开始) |
| size | Integer | 否 | 20 | 每页条数(最大100) |
| sort | String | 否 | id,desc | 排序字段 `字段名,方向` |

#### 排序参数示例
```
?sort=alarmTime,desc          # 按报警时间降序
?sort=alarmLevel,asc          # 按报警级别升序
?sort=createdAt,desc&sort=id,asc  # 多字段排序
```

### 1.4 错误码定义

#### 系统级错误码 (1xxx)
| 错误码 | 说明 | HTTP状态码 |
|--------|------|-----------|
| 200 | 操作成功 | 200 |
| 400 | 请求参数错误 | 400 |
| 401 | 未登录或Token过期 | 401 |
| 403 | 无权限访问 | 403 |
| 404 | 资源不存在 | 404 |
| 405 | 请求方法不允许 | 405 |
| 429 | 请求过于频繁 | 429 |
| 500 | 服务器内部错误 | 500 |
| 503 | 服务暂不可用 | 503 |

#### 认证授权错误码 (11xx)
| 错误码 | 说明 |
|--------|------|
| 1101 | 用户名或密码错误 |
| 1102 | 账号已被禁用 |
| 1103 | 登录失败次数过多，请稍后重试 |
| 1104 | Token无效或已过期 |
| 1105 | 刷新Token无效 |
| 1106 | 验证码错误或已过期 |
| 1107 | 无权限执行此操作 |
| 1108 | 数据权限不足 |

#### 业务错误码 (2xxx-9xxx)
| 错误码 | 说明 | 所属模块 |
|--------|------|----------|
| 2001 | 预警不存在 | 预警中心 |
| 2002 | 预警已处理，不能重复操作 | 预警中心 |
| 2003 | 预警图片不存在 | 预警中心 |
| 3001 | 档案不存在 | 行为档案 |
| 3002 | 档案生成中，请稍后下载 | 行为档案 |
| 4001 | 盒子不存在 | 设备管理 |
| 4002 | 盒子编码已存在 | 设备管理 |
| 4003 | 盒子离线，无法操作 | 设备管理 |
| 4004 | 通道不存在 | 设备管理 |
| 4005 | 通道编码已存在 | 设备管理 |
| 5001 | 用户不存在 | 系统管理 |
| 5002 | 用户名已存在 | 系统管理 |
| 5003 | 手机号已存在 | 系统管理 |
| 5004 | 角色不存在 | 系统管理 |
| 5005 | 角色编码已存在 | 系统管理 |
| 5006 | 不能删除超级管理员 | 系统管理 |
| 5007 | 旧密码错误 | 系统管理 |
| 6001 | 配置项不存在 | 系统配置 |
| 6002 | 配置项不可编辑 | 系统配置 |
| 9001 | 文件上传失败 | 文件服务 |
| 9002 | 文件格式不支持 | 文件服务 |
| 9003 | 文件大小超过限制 | 文件服务 |

#### 错误响应示例
```json
{
  "code": 2001,
  "message": "预警不存在",
  "data": null,
  "timestamp": 1711459200000
}
```

- **URL**: `POST /api/v1/auth/login`

- **请求体**:
```json
{
  "username": "admin",
  "password": "admin123",
  "captcha": "1234",
  "captchaKey": "uuid-string"
}
```
- **响应体**:
```json
{
  "code": 200,
  "message": "登录成功",
  "data": {
    "accessToken": "eyJhbGciOiJIUzI1NiIs...",
    "refreshToken": "eyJhbGciOiJIUzI1NiIs...",

      "id": 1,

  },

```

#### 2.1.2 用户登出
- **URL**: `POST /api/v1/auth/logout`
- **描述**: 用户登出，使当前Token失效
- **请求头**: `Authorization: Bearer {token}`
- **请求体**: 无
- **响应体**:
```json
{
  "code": 200,
  "message": "登出成功",
  "data": null,
  "timestamp": 1711459200000
}
```

#### 2.1.3 刷新Token
- **URL**: `POST /api/v1/auth/refresh`
- **描述**: 使用刷新令牌获取新的访问令牌
- **请求头**: 无特殊要求
- **请求体**:
```json

```

