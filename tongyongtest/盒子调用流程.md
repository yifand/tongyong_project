# Edge-Box 盒子调用流程

> 基于 [Edge-Box] [1.1.3] 接口文档

---

## 一、基础流程骨架

```
设备激活 ──→ 用户登录 ──→ 通道接入 ──→ [人像库准备] ──→ 智能任务配置 ──→ 接收告警结果
  (4.1)       (4.2)       (4.5)         (4.12,仅人脸)     (4.7)           (4.11/4.13/4.14)
```

### 1.1 鉴权前置

所有接口采用统一 **basic + data** JSON 信封格式。除登录/激活/续签外，`basic` 中必须携带 `token`。

### 1.2 密码加密链路

```
原始密码 → MD5 → AES(encrypt_key) → 传输密文
```

- `encrypt_key` 由登录接口返回
- 激活/找回密码场景使用 RSA 公钥加密

## 二、第一阶段：设备激活（4.1）

| 步骤 | 接口 | 说明 |
|------|------|------|
| 1 | `POST /active/status/get` | 查询激活状态，获取 RSA 公钥和加密类型 |
| 2 | `POST /active` | 用公钥加密用户名、密码、密保问题后提交激活 |

---

## 三、第二阶段：用户登录与 Token 维护（4.2）

| 步骤 | 接口 | 说明 |
|------|------|------|
| 3 | `POST /user/login` | 提交用户名 + 加密密码，返回 `token` + `encrypt_key` |
| 4 | `POST /heartbeat` | 每 10s 发送心跳维持 token 有效 |
| 5 | `POST /user/token/renewal` | token 失效（code: -7004）时续签 |

## 四、第三阶段：通道接入（4.5）

| 步骤 | 接口 | 说明 |
|------|------|------|
| 6 | `POST /device/auto_search/list` | 刷新发现局域网内设备 |
| 7 | `POST /camera/add` | 添加通道（RTSP / NET1400 协议，配置视频流地址及鉴权） |

## 五、第四阶段：智能任务配置（4.7）

智能任务支持 **16 种算法类型**：

| disposition_type | 说明 | 分类 |
|-----------------|------|------|
| `face_detect` | 人脸抓拍 | 人脸类 |
| `face_verify_capture` | 抓拍识别 | 人脸类 |
| `face_verify_video` | 视频识别 | 人脸类 |
| `target_detect` | 目标检测（人体/机动车/非机动车/电瓶车/火焰/物品） | 事件类 |
| `target_verify` | 目标识别（预留） | 事件类 |
| `region_enter` | 进入区域 | 事件类 |
| `region_exit` | 离开区域 | 事件类 |
| `region_invade` | 区域入侵 | 事件类 |
| `cross_border` | 越界侦测 | 事件类 |
| `passline_count` | 过线统计 | 事件类 |
| `ebike_enter_elevator` | 电瓶车进电梯 | 事件类 |
| `off_duty_detect` | 离岗检测 | 事件类 |
| `fire_exit_occupied` | 消防通道占用 | 事件类 |
| `safety_helmet` | 未戴安全帽检测 | 事件类 |
| `wear_mask` | 未戴口罩检测 | 事件类 |
| `custom_algoXXX` | 自定义算法 | 事件类 |

### 5.1 配置接口

| 步骤 | 接口 | 说明 |
|------|------|------|
| 8 | `POST /IntelliTask/list` | 查询已有智能任务 |
| 9 | `POST /IntelliTask/update` | 创建/更新智能任务 |
| 10 | `POST /IntelliTask/operation` | 清除统计数据等操作（可选） |

### 5.2 人脸类算法配置要点（需人像库前置）

**前置步骤**（4.12 库与人员管理）：

| 步骤 | 接口 | 说明 |
|------|------|------|
| - | `POST /repositories/create` | 创建人像库（黑名单/白名单/陌生人） |
| - | `POST /face_images/add` | 录入人员照片和身份信息 |
| - | 批量导入 | WebSocket 上传 zip 批量导图（可选） |

**worker_data 配置**：

```
worker_data → face_detect → sensibility     （灵敏度 1-5）
                          → policy          （抓拍策略：模式/间隔/数量/多人模式/优先原则）
                          → pitch           （瞳距，像素）
                          → region          （抓拍区域矩形坐标）
```

**logic_data 配置**：

```
logic_data → face → enabled / role_repo / similarity
                  → global_actions → alarm → upload （上传汇聚平台）
                  → rule[] → type: stranger/white/black
                           → conds → in_repo （布控人像库）
                           → actions （布控行为）
```

### 5.3 事件类算法配置要点（无需人像库）

**worker_data 配置**：

```
worker_data → event_detect → region_event_detect    （区域事件类）
                          → line_event_detect       （过线事件类）
```

**区域事件参数**（region_enter / region_exit / region_invade / ebike_enter_elevator / off_duty_detect / fire_exit_occupied / safety_helmet / wear_mask）：

```
region_event_detect → enabled
                    → event_detect_para[] → id
                                          → region_coordinate（polygon 顶点坐标）
                                          → block_region[]（屏蔽区域）
                                          → detect_target[] → target_type（1-10）
                                                            → enabled
                                                            → max/min_target_size
                                                            → min_duration（最小持续时间）
                                                            → sensitivity
                                          → variant_para（差异参数，如离岗用 post_personnel_info）
                                          → custom_para（自定义算法参数）
```

**目标类型 target_type 枚举**：

| 值 | 目标 |
|----|------|
| 1 | 人脸 |
| 2 | 人体 |
| 3 | 机动车 |
| 4 | 车牌 |
| 5 | 非机动车 |
| 6 | 电瓶车 |
| 7 | 未戴安全帽的人 |
| 8 | 未戴口罩的人 |
| 9 | 火焰 |
| 10 | 物品 |

**过线事件参数**（cross_border / passline_count）：

```
line_event_detect → enabled
                  → event_detect_para[] → id
                                        → line_coordinate（起点/终点坐标）
                                        → direction（left_right / right_left / any_direction）
                                        → block_region[]
                                        → detect_target[]
                                        → custom_para
```

**logic_data 配置**：

```
logic_data → event_detect → service_para → schedule        （排程：按周几+时间段）
                                         → data_statistics  （统计周期：always / daily）
                          → alarm → upload              （上传汇聚平台）
                                  → audio_alarm          （声音告警）
                                  → mail_alarm           （邮件告警）
                                  → popup_screen_alarm   （弹屏告警）
                                  → alarm_out            （告警输出端口）
```

## 六、第五阶段：告警结果接收

### 6.1 实时推送（推荐）

| 方式 | 接口 | 协议 | 说明 |
|------|------|------|------|
| 图片流订阅 | `POST /real_image/subscribe` | WebSocket | 按 face_detect / face_verify / event_detect 类型订阅 |
| 取消订阅 | `POST /real_image/unsubscribe` | WebSocket | 取消指定类型的图片流 |
| 事件通知 | `POST /event/notify` | WebSocket | 通道上下线、网络状态等 |
| 事件订阅 | `POST /event_subscribe/update` | HTTP | 订阅关注的事件类型 |

### 6.2 上传中心推送

| 步骤 | 接口 | 说明 |
|------|------|------|
| 11 | `POST /upload_center/create` | 创建上传地址（通用 / GAT1400 OUT） |
| 12 | `POST /data_upload/update` | 配置推送内容（抓拍图/场景图/比对数据） |
| - | 盒子推送到 `upload_center/data` | JSON 格式，包含 capture + verify + event_list |

### 6.3 事后查询

| 接口 | 说明 |
|------|------|
| `POST /image/list` | 按事件类型/目标属性/时间范围批量查询抓拍记录 |
| `POST /image/detail/by_key/get` | 单张抓拍详情（含结构化属性） |
| `POST /image/capture_statistics/get` | 今日/24h/近7日抓拍统计 |

## 七、人脸 vs 非人脸 流程对比

| 环节 | 人脸算法 | 事件检测算法 |
|------|---------|-------------|
| 激活 | ✓ 4.1 | ✓ 4.1 |
| 登录 | ✓ 4.2.7 | ✓ 4.2.7 |
| 通道接入 | ✓ 4.5 | ✓ 4.5 |
| 人像库准备 | **✓ 4.12 必须** | ✗ 不需要 |
| 智能任务配置 | worker_data: `face_detect`<br>logic_data: `face` | worker_data: `event_detect`<br>logic_data: `event_detect` |
| 配置重点 | 比对谁（人像库 + 阈值 + 黑白名单） | 检测什么目标 + 在哪 + 触发什么动作 |
| 接收告警 | ✓ 4.13/4.11/4.14 | ✓ 4.13/4.11/4.14 |
