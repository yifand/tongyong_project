### 2.1 用户鉴权接口

| 项目 | 描述 |
|------|------|
| **接口地址** | `http://ip:port/security-service/oauth/token` |
| **请求方式** | POST |
| **Content-Type** | `multipart/form-data` |

#### 请求参数

| 参数名 | 类型 | 必填 | 描述 |
|--------|------|------|------|
| `grant_type` | string | 是 | 授权类型（客户端） |
| `client_id` | string | 是 | 客户 ID |
| `client_secret` | string | 是 | 秘钥 |

#### 请求示例

```
grant_type: client_credentials
client_id: 后续对接单独提供
client_secret: 后续对接单独提供
```

#### 返回参数

```json
{
  "access_token": "bf0b73e6-bdc7-4d64-9963-27ccaf61eea4",
  "token_type": "bearer",
  "expires_in": 7200,
  "scope": "order"
}
```

### 2.2 多媒体上传接口

> **重要：** 所有多媒体（图片、录音、视频等）都要先调用此接口再调用业务接口，获取接口返回的 `fileId`，放入各业务接口参数中。

| 项目 | 描述 |
|------|------|
| **接口地址** | `http://ip:port/file-service/resource` |
| **请求方式** | POST |
| **Content-Type** | `multipart/form-data` |

#### 返回示例

```json
[
  {
    "id": "600bde70-7c93-4469-9c0c-a0b753d45eba",
    "name": "微信图片_20210310174103.png",
    "path": "2021/3/19/14"
  },
  {
    "id": "82b70d99-f356-4de8-889c-3a25d0a0b638",
    "name": "bg.png",
    "path": "2021/3/19/14"
  }
]
```

### 2.3 事件上报对接

#### 2.3.1 事件上报

| 项目 | 描述 |
|------|------|
| **接口地址** | `http://ip:port/general-service/api/v1/tasks/report` |
| **请求方式** | POST |
| **Content-Type** | `application/json` |

##### 请求头参数

| 参数名 | 类型 | 必填 | 描述 | 默认值 |
|--------|------|------|------|--------|
| `Authorization` | string | 是 | Bearer Token | 无 |

##### Body 参数

| 参数名 | 类型 | 必填 | 描述 | 默认值 |
|--------|------|------|------|--------|
| `sourceId` | int | 是 | 事件问题来源 | — |
| `categoryLevel1` | string | 是 | 一级分类 | — |
| `categoryLevel2` | string | 是 | 二级分类 | — |
| `categoryLevel3` | string | 是 | 三级分类 | — |
| `categoryLevel4` | string | 是 | 四级分类 | — |
| `contactInfo` | string | 否 | 联系电话 | — |
| `contactPerson` | string | 否 | 联系人 | — |
| `description` | string | 是 | 事件描述 | — |
| `urgencyDegree` | string | 是 | 事件紧急程度（详情见资源接口） | 0（一般） |
| `specialCode` | string | 否 | 事件特殊标签，传编码，多个用逗号隔开（详情见资源接口） | — |
| `address` | string | 是 | 发生地址 | — |
| `poiType` | string | 否 | 兴趣点类型（详情见资源接口） | — |
| `poiCode` | string | 否 | 兴趣点编码（详情见资源接口） | — |
| `coordinate` | JsonObject | 否 | 坐标相关信息 | — |
| `coordinate.coordinateSystem` | string | 否 | 坐标系（wgs84, gcj, bd） | — |
| `coordinate.longitude` | string | 否 | 经度 | — |
| `coordinate.latitude` | string | 否 | 纬度 | — |
| `deviceId` | string | 否 | 设备编号（感知设备时需要填） | — |
| `isStreetAccept` | string | 是 | 是否进街镇核验：1 是 / 0 否 | 0（进区中心核验） |
| `externalSystemCode` | string | 否 | 外系统事件编号 | — |
| `discoveryTime` | Date | 否 | 外系统事件发生时间 | 不传默认为当前时间 |
| `departmentCode` | string | 否 | 立案部门编码（某些特定场景不是根据发生地所在街镇立案的情况，需要传指定立案部门编码） | — |
| `isAutoEnd` | string | 否 | 是否自动结案 | 0 |
| `isDisposeInCitySystem` | string | 是 | 是否需要城运平台流转处置：1 是 / 0 否（针对某些不需要在城运系统流转处理的场景，调事件上报和事件消单结案进行闭环） | 1 |
| `disposition` | JsonObject | 否 | 需要自动派单到固定部门时，填写处置部门相关信息（个别场景需要，默认不填） | — |
| `disposition.mainResponsibility.departmentCode` | string | 否 | 主办部门编码 | — |
| `disposition.mainResponsibility.userId` | string[] | 否 | 主办处理人员 | — |
| `disposition.assistings[].departmentCode` | string | 否 | 协办部门编码 | — |
| `disposition.assistings[].userId` | string[] | 否 | 协办处理人员 | — |
| `attachments` | JsonArray | 否 | 多媒体集合 | — |
| `attachments[].fileGroup` | string | 有媒体时必填 | report（上报）、disposition（处置）、other（其他） | — |
| `attachments[].fileName` | string | 有媒体时必填 | 文件名（带后缀） | — |
| `attachments[].fileId` | string | 有媒体时必填 | 多媒体上传接口中返回的 id | — |

##### 请求示例

```json
{
  "sourceId": 128,

  "categoryLevel2": "99",
  "categoryLevel3": "99",
  "categoryLevel4": "01",
  "contactInfo": "13700000000",

    {
```

#### 2.3.2 事件消单结案

> 外系统派遣至城运平台的案件，可以进行消单，消单后城运平台直接结案。

| 项目 | 描述 |
|------|------|
| **接口地址** | `http://ip:port/general-service/api/v1/tasks/{taskId}/end` |
| **请求方式** | POST |
| **Content-Type** | `application/json` |

##### 请求头参数

| 参数名 | 类型 | 必填 | 描述 | 默认值 |
|--------|------|------|------|--------|
| `Authorization` | string | 是 | Bearer Token | 无 |

> `{taskId}` 为城运系统工单号

##### Body 参数

| 参数名 | 类型 | 必填 | 描述 | 默认值 |
|--------|------|------|------|--------|
| `endNote` | string | 是 | 结案备注 | — |
| `endTime` | Date | 是 | 结案时间 | — |
| `attachments` | JsonArray | 否 | 多媒体集合 | — |
| `attachments[].fileGroup` | string | 是 | report（上报）、disposition（处置）、other（其他） | — |
| `attachments[].fileName` | string | 是 | 文件名（带后缀） | — |
| `attachments[].fileId` | string | 是 | 多媒体上传接口中返回的 id | — |

##### 请求示例

```json
{
  "endNote": "结案归档",
  "endTime": "2021-04-28 14:00:00",
  "attachments": [
    {
      "fileId": "600bde70-7c93-4469-9c0c-a0b753d45eba",
      "fileName": "微信图片_20210310174103.png",
      "fileGroup": "disposition"
    },
    {
      "fileId": "82b70d99-f356-4de8-889c-3a25d0a0b638",
      "fileName": "bg.png",
      "fileGroup": "disposition"
    }
  ]
}
```

### 2.4 事件上报各环节反馈

> **说明：** 新派单系统各节点数据反馈到第三方系统，由第三方公司提供接口。第三方公司接口返回值统一为：

```json
{
  "code": "200",
  "message": "操作成功"
}
```

> `code`: 200 为成功，其他为失败。

#### 2.4.1 核实反馈

派单系统将上报核实相关数据反馈至第三方系统。

##### 接口协议

| 项目 | 描述 |
|------|------|
| **接口地址** | 由第三方提供 |
| **请求方式** | POST |
| **Content-Type** | `application/json` |

##### Body 参数

| 参数名 | 类型 | 必填 | 描述 |
|--------|------|------|------|
| `taskId` | string | 是 | 派单系统工单编号 |
| `externalSystemCode` | string | 是 | 第三方系统工单号 |

| `verificationUser` | string | 是 | 核实人员 |
| `verificationTime` | Date | 是 | 核实时间 |

| 序号 | 接口名称 | 请求方式 | 说明 |
|------|----------|----------|------|
| 1 | 核实反馈 | POST | 上报核实数据反馈 |
| 2 | 核验派单反馈 | POST | 核验派单（自动派单）数据反馈 |
| 3 | 二级平台转派 | POST | 街道城运转派至下级部门 |
| 4 | 接单反馈 | POST | 处置部门/人员接单反馈 |
| 5 | 处置反馈 | POST | 处置信息反馈 |
| 6 | 重新派单 | POST | 平台再次派单反馈 |
| 7 | 结案反馈 | POST | 结案信息反馈 |
| 8 | 预审反馈 | POST | 预审信息反馈 |
| 9 | 核查反馈 | POST | 核查数据反馈 |
| 10 | 归档(作废)反馈 | POST | 归档/作废信息反馈 |

### 统一返回格式

所有第三方回调接口统一返回：

```json
{
  "code": "200",
  "message": "操作成功"
}
```

- `code`: `"200"` 为成功，其他为失败
