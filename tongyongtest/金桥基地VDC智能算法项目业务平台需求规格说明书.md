1. 引言

1.1 项目背景
金桥基地VDC智能算法项目旨在通过视频智能分析技术，实现PDI（出厂前检测）作业规范性监测及违规抽烟行为实时报警。前期已明确由边缘计算盒子在现场接入视频流，实时输出三种基础状态（车门开关、人员存在、进出动作），业务平台负责接收这些状态数据，依据业务规则完成作业合规判定、预警展示、报表生成及系统管理。本文档旨在明确业务平台的功能需求，作为后续设计、开发与验收的依据。
1.2 适用范围
本文档适用于金桥基地VDC智能算法项目业务平台的设计与开发，涵盖预警、报表、权限、日志、设备管理、系统配置等模块。

2. 总体需求
2.1 部署架构
金桥库和凯迪库为两个独立的物理站点，均部署边缘计算盒子，分别接入各自站点的摄像头（PDI工位及抽烟检测区域）。
业务平台建议采用统一部署（如中心服务器），但需实现站点级数据隔离：金桥库的数据仅金桥相关人员可访问，凯迪库的数据仅凯迪相关人员可访问。超级管理员可跨站点访问所有数据。
每个站点的业务独立运行，互不干扰，但平台界面可支持切换站点视图（或通过权限自动过滤）。
视频流不通过平台中转，原始视频存储在各站点的NVR中，平台不提供视频回溯功能（用户需自行登录NVR系统查看）。
2.2 用户角色
超级管理员：拥有所有权限，可配置系统参数、管理用户、查看所有站点的数据。
金桥管理员：仅可查看金桥站点的抽烟检测及PDI检测相关数据，具有该站点的预警查看、报表管理、设备管理等权限。
凯迪管理员：仅可查看凯迪站点的抽烟检测及PDI检测相关数据，具有该站点的预警查看、报表管理、设备管理等权限。
普通查看员：可被分配至特定站点，仅查看该站点的预警和报表，不可进行配置操作（后续扩展）。
2.3 性能要求
实时预警延迟：从事件发生到平台页面展示不超过3秒。
并发用户数：初期支持10个用户同时在线操作。
数据存储周期：报警记录及图片保存至少90天，原始状态流可保存30天用于追溯。
报表查询响应时间：复杂统计查询不超过5秒。
3. 功能模块详述
3.1 预警模块
预警模块分为两个子模块：实时预警预览和历史预警查看。
预警类型包括：抽烟行为报警、PDI作业违规报警。
3.1.1 实时预警预览
功能描述：实时展示当前发生的报警信息，支持声光提示。
界面要求：
页面顶部设消息通知栏，有新报警时闪烁，显示未读数。
点击通知栏展开报警列表，每条报警包含：站点、时间、地点（通道名称）、报警类型（抽烟/PDI违规）、人员面部截图、场景图。
列表支持按站点、类型筛选，按时间排序。
点击单条报警可查看详情，详情页包含：
人员面部大图、场景图
报警发生时间、地点
可下载图片（带Logo水印）
支持标记为“已处理”或“误报”（反馈给算法优化）
声音提示：可配置是否开启声音报警，连接喇叭后自动播放提示音。
3.1.2 历史预警查看
功能描述：查询历史报警记录，支持多维度筛选。
筛选条件：
站点（金桥/凯迪）
时间范围（起始/结束）
报警类型（抽烟/PDI）
通道名称
处理状态（已处理/未处理/误报）
关键词（如工号、姓名等，预留）
展示方式：列表形式，包含与实时预警类似的字段，并提供详情查看按钮。
导出：支持将查询结果导出为Excel或CSV，包含图片链接（或图片文件打包）。
3.2 PDI报表模块
PDI作业以报表形式呈现，非实时干预，主要用于事后回溯与统计分析。
3.2.1 报表生成规则
依据业务规则引擎判定每个工位的作业时长（基于基础状态三元组的时间序列）。
判定结果分为三类：
合格：作业时长 ≥ 标准工时（如12分钟），不生成报警，但可选择性留存记录。
临界：标准工时的90%~~100%（如10分48秒~~12分钟），可配置是否留存。
不合格：作业时长 < 标准工时的90%（如小于10分48秒），必须留存记录并生成报警。
报表内容应包含：
站点信息
车辆信息（如有识别，如车牌或VIN）
工位名称（左前门、左后门、滑移门等）
作业开始时间、结束时间、实际时长
标准工时、偏差值（秒或百分比）
头帧截图（开门瞬间）、尾帧截图（人员离开瞬间）、关键中间帧（可选）
判定结果（合格/临界/不合格）
3.2.2 报表展示与筛选
默认展示最近24小时的不合格记录，可按以下条件筛选：
站点
时间范围
工位
偏差大小（例如只显示偏差大于2分钟的）
Top N（只显示偏差最大的前10条）
列表形式，支持排序，点击可查看详情（包含三张截图）。
支持报表导出（PDF/Excel），可附带截图打包下载。
3.2.3 阈值配置
标准工时可由超级管理员或站点管理员在系统配置中设定（如左前门12分钟、左后门3分钟、滑移门X分钟）。
偏差百分比阈值也可配置（例如设置“不合格”为低于标准工时的90%）。
3.3 权限管理模块
3.3.1 用户管理
支持创建、编辑、删除用户。
用户属性：用户名、密码、姓名、手机号、邮箱、角色、所属站点（可选，用于过滤数据）。
密码策略：可配置密码复杂度、有效期。
3.3.2 角色管理
支持自定义角色，为角色分配功能权限和数据权限。
功能权限：模块访问权限（如预警查看、报表查看、系统配置等）。
数据权限：基于站点的数据隔离（金桥/凯迪/全部）。
初始角色：
超级管理员：所有权限，数据权限为“全部”。
金桥管理员：功能权限为所有模块，数据权限为“金桥”。
凯迪管理员：功能权限为所有模块，数据权限为“凯迪”。
只读用户：功能权限仅为查看预警和报表，无操作权限，数据权限可指定站点。
3.3.3 登录与审计
支持用户名密码登录，可集成SSO（可选）。
登录失败锁定策略（如连续5次错误锁定5分钟）。
所有用户操作记录日志，便于审计。
3.4 日志管理模块
3.4.1 操作日志
记录所有用户的关键操作：登录/登出、配置修改、用户管理、导出数据等。
字段：操作时间、操作用户、IP地址、操作内容、结果（成功/失败）。
支持按时间、用户、操作类型查询，可导出。
3.4.2 系统日志
记录系统运行状态：服务启动/停止、与盒子通信状态、异常错误等。
日志级别（DEBUG/INFO/WARN/ERROR）可配置。
3.4.3 报警日志
记录所有产生的报警，与预警模块数据互通，但独立存储，便于长期保留。
3.5 设备管理模块
管理所有边缘计算盒子及摄像头通道，按站点区分。
3.5.1 盒子管理
盒子列表：显示盒子ID、名称、所属站点、IP地址、状态（在线/离线）、最后心跳时间、版本号。
支持添加/删除盒子（需与盒子激活流程配合）。
支持远程重启盒子、查看盒子资源使用率（CPU/内存/磁盘）。
3.5.2 通道管理
每个盒子关联多个摄像头通道，列表展示通道ID、名称、类型（视频流/抓拍机）、状态、关联算法（抽烟/PDI/两者）。
支持编辑通道参数（如名称、RTSP地址、用户名密码），但需通过盒子接口同步。
支持查看通道实时视频预览（需从盒子取流）。
3.5.3 设备状态监控
大屏展示所有设备在线/离线统计（可切换站点视图）。
异常告警：当盒子离线或通道断流时，在平台产生告警并记录。
3.6 系统配置模块
3.6.1 算法开关
对每个通道或全局，控制是否启用某类算法：
抽烟检测算法
PDI左前门算法
PDI左后门算法
算法参数配置：
抽烟检测：灵敏度、连续触发帧数（防误报）
PDI：标准工时、进出判定时间窗口、人员消失超时阈值等
3.6.2 通用配置
数据留存周期（天）
报警声音开关、音量
邮件/短信推送配置（预留）
报表导出格式、水印设置
3.6.3 业务规则配置
允许超级管理员自定义作业合规判定规则（基于基础状态序列），例如：
进入判定条件：状态7→状态8→状态3
离开判定条件：状态3→状态8→状态7→（状态5或状态1）
提供图形化配置界面或仅后台配置文件支持。
5. 接口需求
5.1 与边缘计算盒子接口
接收盒子推送的实时状态数据（格式见盒子接口文档4.11.2）。
接收盒子推送的报警事件（同格式，event_detect_info）。
调用盒子接口查询通道列表、修改配置、重启等（需封装为平台内部服务）。


| 术语      | 说明                                                            |
| ------- | ------------------------------------------------------------- |
| PDI     | 出厂前检测（Pre-Delivery Inspection）                                |
| 边缘计算盒子  | 部署在现场的AI分析设备，负责视频接入与实时状态识别                                    |
| 基础状态三元组 | 盒子输出的三个布尔值：door_open, person_present, person_entering_exiting |
