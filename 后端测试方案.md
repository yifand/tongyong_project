# PDI智能监测平台 - 后端测试方案

## 1. 测试概述

### 1.1 项目背景
PDI智能监测平台是一个基于视频智能分析的业务平台，实现PDI（Pre-Delivery Inspection）作业规范性监测及违规抽烟行为实时报警。

### 1.2 测试范围

| 模块 | 测试内容 | 优先级 |
|------|----------|--------|
| 认证授权模块 | 登录、登出、Token刷新、权限控制 | P0 |
| 预警中心模块 | 实时预警、历史预警、预警处理 | P0 |
| 行为档案模块 | 档案列表、档案详情、时间线 | P1 |
| 设备管理模块 | 盒子管理、通道管理、实时监控 | P1 |
| 系统管理模块 | 用户管理、角色管理、系统配置、日志审计 | P1 |
| 数据接收服务 | 状态流接收、报警事件接收、心跳接收 | P0 |
| WebSocket服务 | 实时消息推送、状态同步 | P1 |
| 性能测试 | 并发、压力、稳定性 | P2 |
| 安全测试 | 认证绕过、越权访问、SQL注入 | P2 |

### 1.3 测试环境

| 组件 | 版本 | 说明 |
|------|------|------|
| JDK | 17 LTS | Java运行环境 |
| Spring Boot | 3.2.x | 应用框架 |
| PostgreSQL | 16.x | 关系数据库 |
| Redis | 7.x | 缓存/消息队列 |
| Maven | 3.9.x | 构建工具 |

## 4. 性能测试方案

### 4.1 性能测试指标

| 指标项 | 目标值 | 测试方法 |
|--------|--------|----------|
| 登录接口响应时间 | ≤ 500ms | 单用户连续调用10次取平均 |
| 查询接口响应时间 | ≤ 300ms | 单用户查询，数据量1万条 |
| 数据接收处理能力 | ≥ 1000条/秒 | JMeter并发发送状态流 |
| 并发用户数 | ≥ 100 | 100用户同时在线操作 |
| 数据库查询 | ≤ 200ms | 复杂查询执行时间 |
| WebSocket推送延迟 | ≤ 3秒 | 报警产生到前端接收时间 |

## 8. 风险与应对

| 风险 | 影响 | 应对措施 |
