# PDI智能监测平台 - 开发环境搭建指南

## 1. 环境要求概览

| 组件 | 推荐版本 | 最低版本 | 说明 |
|------|----------|----------|------|
| Node.js | 20.x LTS | 18.x | 前端运行环境 |
| JDK | 17 LTS | 17 | 后端运行环境 |
| Maven | 3.9.x | 3.8.x | Java构建工具 |
| PostgreSQL | 16.x | 15.x | 数据库 |
| Redis | 7.x | 6.x | 缓存/消息队列 |
| MinIO | Latest | - | 对象存储(可选) |
| Git | 2.40+ | 2.30+ | 版本控制 |

## 2. 前端开发环境

### 2.1 Node.js安装

#### Windows

1. **下载安装包**
   - 访问 [Node.js官网](https://nodejs.org/)
   - 下载 LTS 版本 (推荐 v20.x)

2. **安装验证**
   ```bash
   node -v
   # v20.11.0

   npm -v
   # 10.2.4
   ```

3. **配置npm镜像(推荐)**
   ```bash
   # 使用淘宝镜像
   npm config set registry https://registry.npmmirror.com

   # 或使用腾讯云镜像
   npm config set registry https://mirrors.cloud.tencent.com/npm/

   # 验证配置
   npm config get registry
   ```

#### macOS

```bash
# 使用 Homebrew 安装
brew install node@20

# 验证
node -v
npm -v
```

#### Linux (Ubuntu/Debian)

```bash
# 使用 NodeSource 安装
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt-get install -y nodejs

# 验证
node -v
npm -v
```

### 2.2 包管理器选择

| 包管理器 | 安装命令 | 特点 |
|----------|----------|------|
| npm | 自带 | 稳定，兼容性好 |
| yarn | `npm install -g yarn` | 速度快，离线模式 |
| pnpm | `npm install -g pnpm` | 节省磁盘空间，推荐 |

```bash
# 安装 pnpm (推荐)
npm install -g pnpm

# 配置pnpm镜像
pnpm config set registry https://registry.npmmirror.com
```

### 2.3 IDE配置

#### VS Code (推荐)

1. **安装扩展**
   - Vue - Official (Vue官方扩展)
   - TypeScript Vue Plugin (Volar)
   - ESLint
   - Prettier
   - Tailwind CSS IntelliSense
   - GitLens

   ```json
     "editor.codeActionsOnSave": {
       "source.fixAll.eslint": "explicit"

     "vue.autoInsert.dotValue": true,
     "vue.complete.casing.tags": "kebab",
     "vue.complete.casing.props": "camel"
   ```

3. **调试配置 (.vscode/launch.json)**
   ```json
   {
     "version": "0.2.0",
     "configurations": [
       {
         "type": "chrome",
         "request": "launch",
         "name": "调试前端",
   ```

```bash

cd pdi-frontend

# 安装依赖 (使用pnpm)

cp .env.development .env.local

pnpm dev

npm run dev
```

## 3. 后端开发环境

### 3.1 JDK安装

#### Windows

1. **下载JDK**
   - 访问 [Oracle JDK](https://www.oracle.com/java/technologies/downloads/) 或 [OpenJDK](https://adoptium.net/)
   - 下载 JDK 17 LTS

2. **安装配置**
   - 运行安装程序，记住安装路径
   - 配置环境变量：

   ```
   JAVA_HOME = C:\Program Files\Java\jdk-17
   PATH 添加 %JAVA_HOME%\bin
   ```

3. **验证安装**
   ```bash
   java -version
   # openjdk version "17.0.8" 2023-07-18

   javac -version
   # javac 17.0.8
   ```

#### macOS

```bash
# 使用 Homebrew 安装
brew install openjdk@17

# 链接到系统
sudo ln -sfn /opt/homebrew/opt/openjdk@17/libexec/openjdk.jdk /Library/Java/JavaVirtualMachines/openjdk-17.jdk

# 添加到 ~/.zshrc
echo 'export PATH="/opt/homebrew/opt/openjdk@17/bin:$PATH"' >> ~/.zshrc

# 验证
java -version
```

#### Linux (Ubuntu/Debian)

```bash
# 安装 OpenJDK 17
sudo apt update
sudo apt install openjdk-17-jdk

# 验证
java -version

# 配置环境变量 (如需)
echo 'export JAVA_HOME=/usr/lib/jvm/java-17-openjdk-amd64' >> ~/.bashrc
echo 'export PATH=$PATH:$JAVA_HOME/bin' >> ~/.bashrc
source ~/.bashrc
```

### 3.2 Maven安装

#### Windows

1. **下载Maven**
   - 访问 [Maven官网](https://maven.apache.org/download.cgi)
   - 下载 apache-maven-3.9.x-bin.zip

2. **解压配置**
   - 解压到 `C:\apache-maven-3.9.x`
   - 配置环境变量：

   ```
   MAVEN_HOME = C:\apache-maven-3.9.x
   PATH 添加 %MAVEN_HOME%\bin
   ```

3. **配置本地仓库**
   修改 `conf/settings.xml`：
   ```xml
   <localRepository>D:/maven-repo</localRepository>

   <!-- 配置阿里云镜像加速 -->
   <mirrors>
     <mirror>
       <id>aliyunmaven</id>
       <name>阿里云公共仓库</name>
       <url>https://maven.aliyun.com/repository/public</url>
       <mirrorOf>central</mirrorOf>
     </mirror>
   </mirrors>
   ```

4. **验证安装**
   ```bash
   mvn -v
   # Apache Maven 3.9.x
   ```

#### macOS

```bash
# 使用 Homebrew
brew install maven

# 验证
mvn -v
```

#### Linux

```bash
# 下载安装
wget https://dlcdn.apache.org/maven/maven-3/3.9.6/binaries/apache-maven-3.9.6-bin.tar.gz
tar -xzf apache-maven-3.9.6-bin.tar.gz
sudo mv apache-maven-3.9.6 /opt/

# 配置环境变量
echo 'export MAVEN_HOME=/opt/apache-maven-3.9.6' >> ~/.bashrc
echo 'export PATH=${MAVEN_HOME}/bin:${PATH}' >> ~/.bashrc
source ~/.bashrc

# 验证
mvn -v
```

### 3.3 IDE配置

#### IntelliJ IDEA (推荐)

1. **安装插件**
   - Lombok
   - MyBatisX
   - Rainbow Brackets
   - GitToolBox
   - .env files support

2. **项目导入**
   - File → New → Project from Existing Sources
   - 选择 `pom.xml` 文件
   - 选择 Import Maven projects automatically

3. **编码配置**
   - File → Settings → Editor → File Encodings
   - Global Encoding: UTF-8
   - Project Encoding: UTF-8
   - Default encoding for properties files: UTF-8

4. **JDK配置**
   - File → Project Structure → Project
   - Project SDK: 选择 JDK 17
   - Project language level: 17

5. **Maven配置**
   - File → Settings → Build → Build Tools → Maven
   - Maven home path: 选择安装的Maven路径
   - User settings file: 选择 conf/settings.xml

6. **代码风格配置**
   - File → Settings → Editor → Code Style → Java
   - 导入项目提供的代码风格文件 (如有)

#### Eclipse

```
1. 安装 Eclipse IDE for Enterprise Java and Web Developers
2. 安装 Lombok 插件
   - 下载 lombok.jar
   - 双击运行，选择 Eclipse 安装目录
3. 配置 JDK: Window → Preferences → Java → Installed JREs
4. 导入项目: File → Import → Existing Maven Projects
```

## 4. 数据库环境

### 4.1 PostgreSQL安装

#### Windows

1. **下载安装**
   - 访问 [PostgreSQL官网](https://www.postgresql.org/download/windows/)
   - 下载 PostgreSQL 16.x 安装程序

2. **安装步骤**
   - 运行安装程序
   - 选择组件: PostgreSQL Server, pgAdmin 4, Command Line Tools
   - 设置超级用户(postgres)密码
   - 端口保持默认 5432
   - 区域设置选择: Chinese (Simplified), China

3. **环境变量**
   ```
   PATH 添加 C:\Program Files\PostgreSQL\16\bin
   ```

4. **验证安装**
   ```bash
   psql --version
   # psql (PostgreSQL) 16.x
   ```

#### macOS

```bash
# 使用 Homebrew
brew install postgresql@16

# 启动服务
brew services start postgresql@16

# 验证
psql --version

# 创建初始数据库
createdb $(whoami)
```

#### Linux (Ubuntu/Debian)

```bash
# 添加PostgreSQL官方仓库
sudo sh -c 'echo "deb https://apt.postgresql.org/pub/repos/apt $(lsb_release -cs)-pgdg main" > /etc/apt/sources.list.d/pgdg.list'
wget --quiet -O - https://www.postgresql.org/media/keys/ACCC4CF8.asc | sudo apt-key add -

# 安装
sudo apt update
sudo apt install postgresql-16 postgresql-client-16

# 启动服务
sudo systemctl start postgresql
sudo systemctl enable postgresql

# 验证
psql --version
```

### 4.2 数据库初始化

#### 创建数据库和用户

```bash
# 切换到postgres用户 (Linux/macOS)
sudo -u postgres psql

# Windows 使用 pgAdmin 或 psql -U postgres
```

```sql
-- 创建数据库
CREATE DATABASE pdi_monitor
    WITH
    ENCODING = 'UTF8'
    LC_COLLATE = 'zh_CN.UTF-8'
    LC_CTYPE = 'zh_CN.UTF-8'
    TEMPLATE = template0;

-- 创建用户
CREATE USER pdi_user WITH PASSWORD 'pdi_password';

-- 授权
GRANT ALL PRIVILEGES ON DATABASE pdi_monitor TO pdi_user;

-- 退出
\q
```

#### 执行初始化脚本

```bash
# 进入项目数据库脚本目录
cd pdi-backend/pdi-admin/src/main/resources/db

# 执行DDL脚本 (Linux/macOS)
psql -U pdi_user -d pdi_monitor -f init-schema.sql

# Windows
psql -U pdi_user -d pdi_monitor -f init-schema.sql

# 或使用 pgAdmin 图形界面执行
```

#### 验证表结构

```bash
# 连接数据库
psql -U pdi_user -d pdi_monitor

-- 查看所有表
\dt

-- 查看表结构
\d sys_user
\d alarm

-- 退出
\q
```

### 4.3 数据库连接配置

编辑 `pdi-admin/src/main/resources/application-dev.yml`：

```yaml
spring:
  datasource:
    type: com.alibaba.druid.pool.DruidDataSource
    driver-class-name: org.postgresql.Driver
    url: jdbc:postgresql://localhost:5432/pdi_monitor
    username: pdi_user
    password: pdi_password
    druid:
      initial-size: 5
      min-idle: 5
      max-active: 20
      max-wait: 60000
```

## 8. 常见问题排查

### 8.1 前端问题

#### 问题: `npm install` 安装失败

**解决方案:**
```bash
# 清除缓存
npm cache clean --force

# 删除 node_modules
rm -rf node_modules package-lock.json

# 重新安装
npm install

# 或使用pnpm
pnpm install
```

#### 问题: 启动后页面空白，控制台报错

**解决方案:**
```bash
# 检查环境变量
# 确认 .env.local 文件存在且配置正确

# 检查后端API是否可访问
curl http://localhost:8080/api/actuator/health

# 检查代理配置 (vite.config.ts)
# 确认 proxy 配置正确
```

#### 问题: ESLint 报错无法提交代码

**解决方案:**
```bash
# 自动修复
pnpm lint:fix

# 或跳过检查 (不推荐用于生产代码)
git commit -m "message" --no-verify
```

### 8.2 后端问题

#### 问题: Maven 下载依赖失败

**解决方案:**
```bash
# 检查网络连接
# 确认settings.xml镜像配置

# 清除本地仓库重新下载
rm -rf ~/.m2/repository
mvn clean install

# 或使用IDE的Maven工具重新导入
```

#### 问题: 启动报错 `Port 8080 was already in use`

**解决方案:**
```bash
# Windows: 查找并结束进程
netstat -ano | findstr :8080
taskkill /PID <PID> /F

# macOS/Linux
lsof -ti:8080 | xargs kill -9

# 或修改端口
# application-dev.yml 中修改 server.port
```

#### 问题: 数据库连接失败

**解决方案:**
```bash
# 检查PostgreSQL是否运行
# Windows: 服务管理器查看
# macOS: brew services list
# Linux: sudo systemctl status postgresql

# 检查连接配置
# 确认用户名密码正确
# 确认数据库已创建

# 测试连接
psql -U pdi_user -d pdi_monitor -h localhost
```

#### 问题: `Flyway` 或 `MyBatis` 报错

**解决方案:**
```bash
# 检查数据库脚本是否执行
# 确认表结构正确

# 查看详细错误日志
tail -f logs/pdi-admin.log

# 重新执行初始化脚本
psql -U pdi_user -d pdi_monitor -f init-schema.sql
```

### 8.3 数据库问题

#### 问题: 初始化脚本执行失败

**解决方案:**
```sql
-- 检查数据库字符集
SHOW SERVER_ENCODING;  -- 应为 UTF8

-- 检查Locale
SHOW LC_COLLATE;  -- 应为 zh_CN.UTF-8

-- 如不正确，重新创建数据库
DROP DATABASE IF EXISTS pdi_monitor;
CREATE DATABASE pdi_monitor
    WITH ENCODING = 'UTF8'
    LC_COLLATE = 'zh_CN.UTF-8'
    LC_CTYPE = 'zh_CN.UTF-8'
    TEMPLATE = template0;
```

#### 问题: 权限不足

**解决方案:**
```sql
-- 授予超级用户权限 (开发环境)
ALTER USER pdi_user WITH SUPERUSER;

-- 或单独授权
GRANT ALL PRIVILEGES ON ALL TABLES IN SCHEMA public TO pdi_user;
GRANT ALL PRIVILEGES ON ALL SEQUENCES IN SCHEMA public TO pdi_user;
```

### 8.4 跨域问题

#### 问题: 前端调用API报 CORS 错误

**解决方案:**
```java
// 后端已配置CORS，检查是否生效
// 查看 SecurityConfig.java 中的CORS配置

// 前端vite.config.ts代理配置
server: {
  proxy: {
    '/api': {
      target: 'http://localhost:8080',
      changeOrigin: true,
      rewrite: (path) => path.replace(/^\/api/, '')
    }
  }
}
```

#### 问题: 前端加载缓慢

**解决方案:**

**解决方案:**
