## 1. 产品概述

### 1.1 产品背景

金桥基地VDC智能算法项目旨在通过视频智能分析技术，实现PDI（Pre-Delivery Inspection，出厂前检测）作业规范性监测及违规抽烟行为实时报警。

### 1.2 产品目标

- **实时监控**: 实现抽烟行为、PDI作业违规的实时检测与预警
- **合规判定**: 基于基础状态三元组实现准确的作业时长判定
- **数据隔离**: 支持金桥库、凯迪库双站点数据独立管理
- **事后追溯**: 提供完整的报表、日志、报警记录查询

### 1.3 核心价值


| 维度   | 价值描述                    |
| ---- | ----------------------- |
| 安全合规 | 杜绝违规抽烟行为，提升作业规范性        |
| 效率提升 | 自动化作业时长统计，减少人工核查        |
| 管理透明 | 全流程数据记录，支持事后追溯分析        |
| 扩展灵活 | 基于状态三元组的规则引擎，支持业务规则灵活调整 |

## 3. 功能模块设计

### 3.1 预警中心模块

#### 3.1.1 实时预警预览

**功能描述**: 实时展示当前发生的报警信息，支持声光提示

**界面要素**:

```
┌─────────────────────────────────────────────────────────────┐
│                                                            │
├─────────────────────────────────────────────────────────────┤
│ 筛选: [站点▼] [类型▼] [时间▼]           [声音开关 🔊]         │
├─────────────────────────────────────────────────────────────┤
│ 报警列表                                                   │
│ ┌──────┬──────────┬─────────┬──────────┐ │
│ │ 站点 │ 时间     │ 地点    │ 类型     │ │
│ ├──────┼──────────┼─────────┼──────────┤ │
│ │ 金桥 │ 10:23:15 │ 通道a   │ PDI违规  │ │
│ │ 金桥 │ 10:20:08 │ 通道b   │ 抽烟     │ │
│ └──────┴──────────┴─────────┴──────────┘ │
└─────────────────────────────────────────────────────────────┘
```

**功能要点**:

- 新报警自动推送，消息栏闪烁提醒
- 声音开关控制报警提示音
- 报警卡片展示面部截图和场景截图（并排）
- 操作：查看详情、处理报警

#### 3.1.2 历史预警查看

**功能描述**: 查询历史报警记录，支持多维度筛选和导出

**筛选条件**:

- 站点：金桥/凯迪（单选/多选）
- 时间范围：起止时间选择器
- 报警类型：抽烟/PDI违规/全部
- 通道名称：下拉选择

**列表字段**:

| 字段  | 说明     | 示例值                 |
| --- | ------ | ------------------- |
| 站点  | 报警发生站点 | 金桥/凯迪               |
| 时间  | 报警发生时间 | 2026-03-25 10:23:15 |
| 地点  | 通道/位置  | 左前门/休息区A            |
| 类型  | 报警类型   | PDI违规/抽烟            |
| 操作  | 查看详情   | [查看]按钮              |

**操作功能**:

- **查看**: 点击弹出报警详情弹窗，展示完整报警信息和图片

---

#### 3.1.3 预警详情弹窗

**弹窗触发方式**: 点击实时预警或历史预警列表中的[查看]按钮

**弹窗布局**:

```
┌─────────────────────────────────────────────────────┐
│ 报警详情                                      [×]  │
├─────────────────────────────────────────────────────┤
│                                                     │
│  ┌──────────────────┬───────────────────────────┐  │
│  │                  │  时间: 2026-03-25          │  │
│  │                  │       10:23:15           │  │
│  │   [报警图片]      │                           │  │
│  │   (面部/场景图)   │  地点: 金桥 - 左前门        │  │
│  │                  │                           │  │
│  │                  │  类型: PDI违规             │  │
│  │                  │                           │  │
│  │                  │                           │  │
│  └──────────────────┴───────────────────────────┘  │
└─────────────────────────────────────────────────────┘
```

**左侧图片区**:

- 显示报警时的抓拍图片（面部截图和场景截图可切换或并排展示）
- 支持点击图片放大查看
- 支持下载当前图片

**右侧信息区**:

| 字段     | 说明               | 示例值                 |
| -------- | ------------------ | ---------------------- |
| 时间     | 报警发生时间       | 2026-03-25 10:23:15    |
| 地点     | 站点和具体位置     | 金桥 - 左前门          |
| 类型     | 报警类型及相关信息 | PDI违规 / 抽烟         |

- PDI违规类型额外显示：实际时长、标准时长、偏差时长
- 抽烟类型额外显示：检测置信度

**底部操作区**:

- 标记已处理：将报警状态更新为已处理
- 标记误报：将报警标记为误报，不计入统计
- 下载图片：下载报警截图（面部+场景）

**弹窗交互**:

- 点击遮罩层或右上角[X]关闭弹窗
- 支持ESC键关闭
- 图片支持缩放、拖拽查看

### 3.2 行为档案模块

#### 3.2.1 功能概述

行为档案模块用于展示和管理PDI作业检测过程中产生的所有作业记录，提供从作业开始到结束的完整时间线追溯功能。

#### 3.2.2 核心字段定义

**行为档案记录字段**:


| 字段     | 字段名                | 数据类型     | 说明                      |
| ------ | ------------------ | -------- | ----------------------- |
| 开始时间   | start_time         | DateTime | 检测到人员进入车内的时间（状态S3触发时刻）  |
| 结束时间   | end_time           | DateTime | 检测到人员离开的时间（状态S5/S1触发时刻） |
| 预估检查时间 | estimated_duration | Integer  | 该工位配置的标准工时（分钟）          |


#### 3.2.3 行为档案列表功能

**列表展示字段**:


| 字段     | 说明               | 示例值                     |
| ------ | ---------------- | ----------------------- |
| 开始时间   | 作业开始时间戳          | 2026-03-25 10:00:00     |
| 结束时间   | 作业结束时间戳，进行中显示"—" | 2026-03-25 10:08:32 或 — |
| 预估检查时间 | 配置的标准工时          | 12分钟                    |
| 状态     | 作业状态：进行中显示"进行中"；已完成则根据是否达标显示"达标"或"未达标" | 进行中 / 达标 / 未达标                 |
| 操作     | 查看详情入口           | [详情]按钮                  |


**筛选条件**:

- 站点：金桥 / 凯迪 / 全部
- 状态：达标 / 未达标 / 全部
- 时间范围：今天 / 最近7天 / 最近30天 / 自定义日期范围

**排序规则**：默认按开始时间倒序（最新的在前）

#### 3.2.4 行为档案详情弹窗（时间线视图）

**弹窗结构**：

- **头部**：标题"行为档案详情" + 关闭按钮
- **基本信息区**：展示该档案的关键信息
- **时间线区**：竖向时间线展示完整作业过程
- **底部操作区**：下载图片包、关闭按钮

**基本信息区字段**:


| 字段     | 说明                            |
| ------ | ----------------------------- |
| 开始时间   | 2026-03-25 10:00:00           |
| 结束时间   | 2026-03-25 10:08:32（进行中显示"—"） |
| 预估检查时间 | 12分钟                          |
| 实际检查时间 | 8分32秒（进行中显示"—"）               |
| 站点     | 金桥库 / 凯迪库                     |
| 工位     | 左前门 / 左后门 / 滑移门               |


**时间线元素定义**:

每个时间节点包含以下要素：


| 元素  | 字段名       | 说明                         |
| --- | --------- | -------------------------- |
| 时间点 | timestamp | 状态变化或关键动作发生的精确时间（HH:MM:SS） |
| 图片  | image_url | 该时间点对应的视频截图，点击可放大查看        |
| 行为  | action    | 该时间点发生的动作描述                |


**标准时间线节点**（按作业流程）：


| 顺序  | 时间点  | 行为描述      | 图片内容        |
| --- | ---- | --------- | ----------- |
| 1   | 作业开始 | 人员进入车内    | 门开瞬间，人员进入画面 |
| 2   | 作业中  | 检查作业进行中   | 人员在车内作业画面   |
| 3   | 作业结束 | 人员离开，检查结束 | 人员离开瞬间画面    |


**时间线交互**：

- 图片支持点击放大查看
- 时间线支持滚动（节点较多时）
- 支持下载该档案的所有截图（图片包）

#### 3.2.5 数据关联

行为档案记录与以下数据实体关联：

- **Channel（通道）**：记录产生档案的摄像头通道
- **PDITask（PDI作业）**：档案对应的作业记录
- **StateStream（状态流）**：时间线节点的数据来源

## 7. 核心规则引擎

### 7.1 状态机定义

**基础状态三元组**:

```
S = (door_open, person_present, person_entering_exiting)

状态编号映射:
S1 = (0, 0, 0) - 空闲
S3 = (0, 1, 0) - 门关+有人
S5 = (1, 0, 0) - 门开+无人
S7 = (1, 1, 0) - 门开+有人+未进出
S8 = (1, 1, 1) - 门开+有人+进出中
```

### 7.2 进入判定规则

```
触发条件: 检测到状态序列 [S7 → S8 → S3]

时间线示意:
─────────────────────────────────────────────────→
    │      │      │
    S7     S8     S3
   (准备) (进出) (进入完成)
    │      │      │
    └──────┴──────┘
       人员进入车内

记录: entry_time = S3.timestamp
开始累积车内作业时间
```

### 7.3 离开判定规则

```
触发条件: 检测到状态序列 [S3 → S8 → S7 → (S5或S1)]

时间线示意:
─────────────────────────────────────────────────→
    │      │      │       │
    S3     S8     S7      S5/S1
   (车内) (进出) (已出)   (离开)
    │      │      │       │
    └──────┴──────┴───────┘
        人员离开流程

记录: exit_time = S5/S1.timestamp
结束作业计时
```

### 7.4 作业时长计算

```
作业时长 = exit_time - entry_time

判定:
IF duration >= standard_time THEN 合格
IF duration >= 0.9 * standard_time THEN 临界
IF duration < 0.9 * standard_time THEN 不合格(报警)
```

### 7.5 多车门合并流程

```
┌────────────────────────────────────────────────────────┐
│                    车辆PDI作业                          │
│  ┌─────────────┐    ┌─────────────┐    ┌─────────────┐│
│  │  左前门作业  │    │  左后门作业  │    │  滑移门作业  ││
│  │  start: T1  │    │  start: T2  │    │  start: T3  ││
│  │  end:   T4  │    │  end:   T5  │    │  end:   T6  ││
│  └──────┬──────┘    └──────┬──────┘    └──────┬──────┘│
│         │                  │                  │       │
│         └──────────────────┼──────────────────┘       │
│                            ↓                         │
│              总时长 = max(T4,T5,T6) - min(T1,T2,T3)   │
│                            │                         │
│                     判定: 是否≥15分钟                  │
└────────────────────────────────────────────────────────┘
```
